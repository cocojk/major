2. Compile the program without optimization

1. Measure the size of the executtable file

2. Measure the execution time

 

Hint for Windows

Watch out compilers°Ø default compile options.

Compilers are often set to use some speed optimizations as a default compiler option. So you have to deselect them manually.

 

Ref.

* Microsoft Visual Studio Environment : MSDN (Compiler references)

  - °∞/Od°± option disables optimizations

* Intel C/C++ Compiler : Intel C/C++ Compiler User Guide (Optimization section)

  - °∞-Od°± option disables optimizations

For procedure call traces, at first you should understand calling convention on your system and then you can confirm it with GDB. You can also use the pstack utility for call traces, but with pstack utility you can only get a very limited amount of information.

Ref.

 * Debugging with GDB

- http://sources.redhat.com/gdb/download/onlinedocs/gdb.html

* Kaashoek's  note on GCC°Øs calling convention on the x86

Ref.

* ELF Specification document – Section 2. Program Loading and Dynamic Linking

   - http://www.skyfree.org/linux/references/ELF_Format.pdf

 * Unix man page

   - Just type °∞man pmap°± on command line.

6. Determine the dynamic execution frequencies of code sections using a profiling tool.

 

Hint for Windows

Visual studio profiler

Check profile option (Enable Profile) on the Project settings/Link tab. Click profile command on the build menu. You can choose 5 types of profiling and if you use line coverage profiling you can check whether each line is called or not. At this time I°Øm looking for a profiling tool that shows profiles about heap and global variables usage in windows environment.

Function timing profiling reveals spent time for each procedure and also hit count. But I think can try to find a better profiling tool for windows.

 

Hint for SUN OS(or Linux)

Use the gprof utility as a profiling tool. It can analyze function execution count and time, even execution count of individual source code line.

Ref.

 * GNU gprof manual

- http://www.cs.utah.edu/dept/old/texinfo/as/gprof.html
